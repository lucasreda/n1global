## Autenticação

Todas as requisições exigem um Bearer Token via cabeçalho de Autorização, que é único para cada loja. Este token está disponível para o proprietário/administrador da loja em seu painel.

Todas as requisições disponíveis em nossa API seguem o prefixo: `https://accounts.cartpanda.com/api/{store-slug}/`

### Bearer Token

No canto superior direito do painel da sua loja, clique no ícone de perfil e você poderá copiar o token no menu suspenso.

### Store-slug

O store slug é o nome exclusivo da sua loja fornecido pela Cartpanda. Por exemplo, se o domínio da sua loja for `example-test.mycartpanda.com`, então o subdomínio `example-test` é o seu store slug exclusivo e você deve usá-lo aqui ao fazer uma chamada de API.

## Carrinho Abandonado

### Listar Carrinhos Abandonados

- **Método:** GET
- **URL:** https://accounts.cartpanda.com/api/{shop-slug}/abandoned
- **Descrição:** Obtém uma lista de todos os carrinhos abandonados da loja após 30 minutos.
- **Autenticação:** Bearer Token (Authorization: Bearer 123)
- **Parâmetros de Path:**
  - `shop-slug` (string, obrigatório): O slug único da sua loja.
- **Parâmetros de Query:**
  - `created_at_max` (string, opcional): Intervalo máximo de criação (formato: AAAA-MM-DD HH:MM:SS).
  - `created_at_min` (string, opcional): Intervalo mínimo de criação (formato: AAAA-MM-DD HH:MM:SS).
  - `since_id` (integer, opcional): Obtém registros após este ID.
  - `updated_at_max` (string, opcional): Intervalo máximo de atualização (formato: AAAA-MM-DD HH:MM:SS).
  - `updated_at_min` (string, opcional): Intervalo mínimo de atualização (formato: AAAA-MM-DD HH:MM:SS).
- **Exemplo de Requisição (cURL):**
```bash
curl --request GET \
  --url https://accounts.cartpanda.com/api/{shop-slug}/abandoned \
  --header 'Accept: application/json, application/xml' \
  --header 'Authorization: Bearer 123'
```

}

      "quantity": 2

      "variant_id": 300852,
